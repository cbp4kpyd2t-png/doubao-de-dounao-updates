# 豆包的豆脑

## 差异化创意模式（v1.3）

- 默认启用“效率优先智能模式”：从原 TXT 中安全提取商品事实，不覆盖、删除或重命名原文件。
- 每件商品自动规划 50 张复合角度不同的创意图；每轮向 ChatGPT 固定提交 5 张彼此独立、都可作为 Temu 主图的任务。
- 允许豪华场景、丰富道具、人物使用或人物在商品旁边，但商品始终必须完整、醒目且一眼可见。
- 疑似模板复制错误会被忽略；数量等互相冲突的事实不会擅自选择，以参考图为准。
- 管理文件保存在每个商品的 `豆脑配置` 目录；单个商品的额外要求可写入商品根目录的 `创意要求.txt`。
- UI 中的智能模式开关和本批次补充要求会自动记忆；关闭智能模式可使用旧版 TXT 直发流程。

Windows 桌面程序，直接操作当前已经打开的 Microsoft Edge 中的 ChatGPT 网页，为产品文件夹批量生成并保存电商主图。程序不会启动新浏览器或创建独立浏览器配置。

## 输入目录

```text
产品总目录/
  产品A/
    参考图1.jpg
    参考图2.png
    提示词.txt
  产品B/
    reference.webp
    prompt.txt
```

每个一级子文件夹是一个产品。程序按文件名顺序上传全部 PNG/JPG/JPEG/WEBP，并合并全部 TXT 内容。`outputs` 会自动排除，不会被当作产品。

## 使用

1. 安装依赖：`npm install`
2. 开发运行：`npm start`
3. 先自行打开 Microsoft Edge，进入并登录 `https://chatgpt.com/`，再点击“连接当前 Edge”。
4. 选择产品总目录并开始任务。
5. 构建安装包：`npm run dist`
6. 构建可分发目录：`npm run pack`；生成带 SHA256 清单的升级包：`npm run release:update`。

输出位于所选目录的 `outputs`。每次新任务创建 `陛下请查收_1`、`陛下请查收_2` 等总目录，各商品图片集中放入同名子目录；`task-state.json`、`index.csv` 和 `run.log` 保存在 `outputs` 根目录。任务可暂停、停止并从断点恢复。

生成完成后，程序会打开图片查看器，对当前大图使用右键菜单“将图像另存为”，再按右侧缩略图从上到下逐张切换保存。文件直接写入对应产品输出目录并执行解码与重复校验。

## 效率调度、恢复与质量

- “效率优先自适应”会学习近期真实生成耗时。连续成功时靠近用户设置的最短间隔；失败率升高时逐步放慢；遇到请求频繁时进入持久化冷却，冷却结束后从断点继续。
- 每个上传、发送、生成、确认、保存和质量步骤都有独立截止时间、重试次数、恢复动作和持久化记录。程序意外退出后会将未完成步骤标记为中断，再从安全步骤继续。
- 图片质量检测覆盖解码、重复、尺寸、方图、白底边缘和与参考图的颜色结构相似度。默认以产量优先：损坏、重复、过小和明显非方图会拒绝；白底与主体相似度只提示。可在界面开启严格白底或严格主体一致。

## 多电脑安全更新

默认更新源为公开仓库 `cbp4kpyd2t-png/doubao-de-dounao-updates`。发布新版时，把 `dist/update/update-manifest.json` 提交到仓库主分支，并把 ZIP 作为同版本 GitHub Release 附件上传。软件只接受更高版本，下载后必须通过清单中的 SHA256 校验；任务运行期间不会更新。安装失败会自动恢复旧程序，界面设置、管理员授权和任务断点保存在用户数据目录，不会被覆盖。

## 安全与限制

- 程序不会保存密码，也不会处理验证码或绕过站点安全检查。
- 自动化使用前台可访问控件和键盘输入；运行期间不要操作目标 Edge 窗口、鼠标或键盘。
- ChatGPT 网页结构可能更新；一般页面故障会自动返回 ChatGPT 并开启新对话，验证码、安全检查或人工登录仍需用户正常处理。
- 主体一致性使用本地轻量颜色结构启发式，只用于自动初筛，不代替人工视觉质检。
